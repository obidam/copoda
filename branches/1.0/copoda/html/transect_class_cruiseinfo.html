COPODA "transect" class: 'cruise_info' property

This property is itself a class similarly called cruise_info. I acknowledge that the use of similar names for a Transect object property and its inner content as another class may be misleading. To avoid any confusion between the Transect property and its value (both having the same name here), we'll color in blue the reference to the Transect property and in red the reference to its value as a (sub)-class. The cruise_info class has the following properties:


Properties Type Description
NAME string Name of the cruise
PI_NAME string Name of the Principal Investigator (PI) of the cruise
PI_ORGANISM string PI's affiliation
SHIP_NAME string Name of the ship
SHIP_WMO_ID string WMO identifier of the ship. They can be found here: http://www.wmo.int/pages/prog/amp/mmop/wmo-number-rules.html
DATE datenum (2x1 array) First and last days of the cruise, as return by datenum
N_STATION double Number of stations/profils performed during the cruise


The use of another class instead of a classic structure to gather these informations in a Transect object was motivated by the eventual possibility of adding exotic additional informations or manipulating string contents (although not exploited at this time). To the end user, this doesn't make any difference. 


To fill informations on a cruise_info object you can call the default constructor and modify fields afterward:
>> C = cruise_info
===== Cruise_info object content description:
           CRUISE INFORMATION(S)
                      Date: From Jan. 00 0000 to Jan. 00 0000 (0 days)
      Number of station(s): 0
>> C.NAME = 'Hypothetic cruise'
===== Cruise_info object content description:
           CRUISE INFORMATION(S)
                      Name: Hypothetic cruise
                      Date: From Jan. 00 0000 to Jan. 00 0000 (0 days)
      Number of station(s): 0


Or simply specify fields properties when calling the constructor:
>> C = cruise_info('NAME','Hypothetic cruise','N_STATION',123)
===== Cruise_info object content description:
           CRUISE INFORMATION(S)
                      Name: Hypothetic cruise
                      Date: From Jan. 00 0000 to Jan. 00 0000 (0 days)
      Number of station(s): 123


Note that all cruise_info properties are in capital letters !
Now, coming back to the cruise_info property of a Transect object, a simple workflow to specify it could be:
>> T = transect;
>> T.cruise_info = cruise_info('NAME','Hypothetic cruise','DATE',datenum(2006,5,[1 15],0,0,0));

From the previous statements, we assigned to the Transect object cruise_info property, a cruise_info class instance. Note however, that you can also access or modify a property of the cruise_info class instance from a direct reference from the cruise_info Transect object property like:

>> T.cruise_info.N_STATIONS = 12
